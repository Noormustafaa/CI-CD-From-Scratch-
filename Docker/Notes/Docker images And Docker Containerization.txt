---> Docker Containers ?
What is Docker Containers?

A Docker Container is a running instance of a Docker Image!
A Docker Image is a blueprint, and a Docker Container is the actual running application created from that blueprint.

Simple Definition

A Docker Container is an isolated environment that contains an application along with all the files, libraries, dependencies, and configurations required to run it.

Each container runs independently from other containers and from the host system:

Means- once we create Docker image! a Blue print now for run that project in actual we need Docker container!




-->Docker Containers are Isolated, Lightweight, and Portable?

Isolated 

Meaning: A container lives in its own separate environment and does not interfere with other containers or your computer!
ex:

Computer
│
├── Container A (Java 17)
│
├── Container B (Python 3.12)
│
└── Container C (MySQL)




Each container has its own environment.

Example

Your laptop has:

Java 21

But inside a Docker container:

Java 17

The container uses Java 17 without affecting Java 21 installed on your computer.:

That's why we call it isolated.


--2:Lightweight 

Meaning: Containers use very little memory and storage compared to Virtual Machines (VMs).

--3:Portable 

Meaning: You can move a container from one computer to another and it will work the same way.



--Docker containers can be started, stopped, and deleted.


-->fundamental workflow of Docker.

1. Dockerfile (The Recipe)
The Dockerfile is a simple text file containing a list of step-by-step instructions. It tells Docker what operating system to use, what dependencies to install, and how to run your application.

Real-world Example: A recipe card that says: "Take a bowl, add 2 eggs, add 1 cup of flour, and bake at 180°C."

Docker Example: A file that says: "Take Ubuntu Linux, install Python, copy my app.py file, and start it.

2. Docker Image (The Cake Mix)
When you tell Docker to "Build" your Dockerfile, it creates a Docker Image. This is a frozen, unchangeable (read-only) package containing everything your application needs to run. It doesn't do anything yet; it’s just sitting there on your computer.


Real-world Example: A boxed cake mix you buy from the store. It has all the ingredients perfectly measured and mixed together, sitting on your shelf waiting to be used.

Docker Example: A template package of your app that you can share with your team or upload to GitHub.


3. Docker Container (The Actual Cake)
When you tell Docker to "Run" an image, it becomes a Docker Container. This is the living, breathing, running instance of your application. You can interact with it, open it in your browser, or send data to it.

Real-world Example: The actual, baked cake that you just took out of the oven and can now eat.

Docker Example: Your website is live and users can now visit it.
